# 规则系统上下文

本上下文定义 LanJing 如何通过规则获取、转换和增量提供标准媒体资源。这里只记录规则系统与产品层之间稳定的领域语言和契约，不记录具体代码实现或临时方案。

## Language

**Typed Flow Core（类型化流程核心）**：
规则系统中与媒体类型无关的执行语义。它处理类型化节点之间的控制流、错误、追踪和流式执行，不理解来源专有术语，也不负责前端展示。

**Standard Intent（标准意图）**：
规则对外暴露的稳定可调用入口，例如 `Search`、`Discover`、`ResolveItem`、`ListUnits`、`ResolveAsset` 和 `ContinueAction`。意图表达调用方想做什么，不决定产品界面结构。

**Controlled Mapper（受控映射器）**：
规则层把已解析来源数据转换为标准媒体模型的映射语义。它使用受控映射节点、字段映射、默认值和安全转换，不允许来源规则直接定义产品界面。

**Mapper Vocabulary（映射词汇）**：
受控映射器及其上下游共享的稳定字段键、模板变量和 marker 词汇。它只承载跨模块稳定契约，例如 `title`、`url`、`bookUrl`；来源专属协议字段例如 `vod_*` 保留在来源模块自己的词汇空间中。

**Media Resource Graph（媒体资源图）**：
LanJing 消费的标准化产品数据图。它包含来源画像、媒体条目、集合、消费单元、载荷资源、关系、动作和展示提示，并要求资源具备稳定 ID 与来源归属。

**Media Resource Graph Delta（媒体资源图增量）**：
对媒体资源图的一次增量更新。增量可以创建资源、补全已有资源、增加关系、挂载载荷资源或添加后续动作，并且必须能按稳定的来源归属 ID 确定性合并。

**Source-owned Resource ID（来源持有资源 ID）**：
由来源归属和来源侧稳定 key 共同构成的可逆资源标识。它让 `MediaItem`、`MediaUnit`、后续意图输入以及资源图增量合并指向同一实体，同时避免把原始来源 URL 或临时上下文泄露成调用方契约。

**SourceProfile（来源画像）**：
来源身份与能力记录。它描述来源名称、图标、规则版本、风险提示、支持的标准意图，以及帮助应用安全展示和执行规则的其他来源级事实。

**MediaItem（媒体条目）**：
主要媒体实体，例如书籍、电影、专辑、播客、漫画、文章、课程、本地文件或混合媒体条目。共享字段保留在基础条目上，媒体类型专属信息放入明确的类型化元数据中。

**MediaCollection（媒体集合）**：
媒体资源的容器或分组，例如榜单、书架、播放列表、频道、专题、系列、作者页、来源目录或混合集合。

**MediaUnit（媒体单元）**：
隶属于媒体条目或集合的可排序消费单元，例如章节、剧集、曲目、漫画页、文章段落或课程小节。媒体单元表达顺序与进度，不直接等同于可消费载荷本身。

**MediaAsset（媒体载荷）**：
挂载到媒体条目或媒体单元上的可消费载荷，例如正文、图片、音频流、视频流、字幕、封面或附件。载荷可以晚于其父级媒体单元再解析得到。

**MediaRelation（媒体关系）**：
媒体资源之间的类型化关系，例如作者、属于某系列、包含、上一项、下一项、相似内容、同专辑、同频道或来源归属。

**MediaAction（媒体动作）**：
资源暴露的后续动作，例如继续加载、选择分类、切换线路、选择清晰度、解析载荷或打开外部页面。动作携带不透明执行载荷，不向前端泄露内部流程节点。

**PresentationHint（展示提示）**：
附着在标准媒体资源上的低优先级展示提示，例如封面比例、主色、密度偏好、模板族建议或排序提示。它可以辅助模板选择，但不能定义布局、组件或自定义 UI。

**Rule Blueprint View（规则蓝图视图）**：
围绕媒体意图、标准输出、映射器完整度和来源能力组织的高层规则编辑视图。它是同一份可执行规则的编辑投影，不是独立的运行时模型。

**Flow View（流程视图）**：
暴露真实类型化节点、端口、边、控制流和映射节点的低层规则编辑视图。它与蓝图视图绑定到同一份可执行规则，用于调试和高级编辑。

## 边界

- 规则通过 `Standard Intent` 输出标准媒体资源、关系、动作和有限展示提示，不定义 UI 布局、组件或来源专属页面结构。
- `Controlled Mapper` 负责来源数据到标准媒体模型的受控转换；来源专属协议字段不进入共享映射词汇。
- 资源必须具备稳定来源归属 ID，`Media Resource Graph Delta` 必须可确定性合并。
- `PresentationHint` 只能辅助产品模板选择和排序，不能成为来源控制界面的通道。
- 产品体验只依赖标准媒体模型，不依赖规则内部流程节点或来源专属协议。
